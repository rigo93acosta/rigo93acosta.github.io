## Git Clone 

> [!tip] Git clone con todas las ramas
### Comando b√°sico

```bash

```

Este comando descarga una copia exacta del repositorio con **todas las ramas**, **tags**, y el **historial completo**. Es √∫til si quieres replicar el repositorio como est√°, incluyendo todas las referencias. Sin embargo, esto clonar√° el repositorio en modo "bare", lo que significa que no podr√°s trabajar directamente en √©l (no se ver√° como un directorio de trabajo normal de Git).
### Para clonar normalmente pero con todas las ramas

Si deseas clonar el repositorio de forma que puedas trabajar en √©l y, adem√°s, traerte todas las ramas y el historial, usa:

```bash

```
#### Detalles de las opciones
1. **Clonar todo el historial y las ramas**:
   - El comando `git clone <URL>` por defecto ya clona el historial completo y la rama por defecto (generalmente `main` o `master`).
   - Para asegurarte de traer **todas las ramas remotas**, ejecuta despu√©s:
     ```bash
     git fetch --all
     ```
     Esto traer√° todas las referencias y ramas del repositorio remoto.
2. **Ver todas las ramas locales y remotas**:
   Despu√©s de clonar y hacer `git fetch --all`, puedes listar todas las ramas disponibles:
   ```bash
   git branch -a
   ```
   Esto te mostrar√° tanto las ramas locales como las remotas.
3. **Traer todas las ramas remotas a locales**:
   Si deseas tener todas las ramas remotas tambi√©n como ramas locales en tu repositorio, puedes ejecutar:
   ```bash
   for remote in `git branch -r`; do git branch --track ${remote#origin/} $remote; done
   git fetch --all
   git pull --all
   ```
Este conjunto de comandos clonar√° el repositorio, traer√° todas las ramas y referencias remotas, y replicar√° esas ramas en tu repositorio local. Tendr√°s disponible tanto el historial de commits como las ramas.
#### Para descargar tambi√©n los tags:

Si necesitas asegurarte de traer los **tags** junto con el historial de commits, puedes hacerlo as√≠:
```bash

```
#### Clonar sin limitar el historial (para repositorios grandes):
Si el repositorio es muy grande, algunos clonan con un l√≠mite de historial usando `--depth`. Para evitar esto y clonar todo el historial, no uses la opci√≥n `--depth`.
